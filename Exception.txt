-> It is an unwanted or unexpected event that occurs during the execution of a program and disrupts the normal flow of the code.
It represents an error condition ‚Äî like dividing by zero, accessing a null object, reading a missing file, etc.

Exception Handling -> in Java is a mechanism that allows developers to handle runtime errors gracefully, ensuring that the program doesn't crash unexpectedly. It helps in maintaining the normal flow of the application, provides meaningful error messages, and separates error-handling logic from the main business logic.

Checked Exceptions -> are those that the compiler forces you to handle, usually because they might be recoverable (like IOException or SQLException). You must either use a try/catch block or declare them with throws.

Unchecked Exceptions -> on the other hand, are usually due to programming errors and are not checked at compile time. They include exceptions like NullPointerException and ArithmeticException, and the compiler does not force you to handle them.

how many try and catch we can write in one problem?
###################################################
‚ÄúThere is no fixed limit. We can write multiple try-catch blocks in a program.
Inside a single try block, we can also have multiple catch blocks to handle different exceptions.‚Äù

‚ÄúAlthough Java allows multiple try-catch blocks, in real projects we avoid excessive try-catch and prefer global exception handling (like @ControllerAdvice in Spring Boot) to keep code clean.‚Äù

When we get NullPointerException?
###################################
‚ÄúA NullPointerException occurs when we try to access or operate on an object reference that is null.
In other words, the variable is not pointing to any object, but we are trying to use it.‚Äù  
To avoid this we use Optional and  custom exceptions like ResourceNotFoundException.‚Äù

How you handle Exceptions in SpringBoot?
#########################################
In Spring Boot, we handle exceptions using centralized exception handling.
We create custom exceptions and handle them globally using @RestControllerAdvice with @ExceptionHandler.
This allows us to return proper HTTP status codes and meaningful error responses instead of stack traces.
