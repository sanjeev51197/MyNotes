
What is Spring boot
################################

Spring Boot is an open-source Java-based framework used to create stand-alone applications quickly and easily with minimal configuration.

Spring Boot Stand-Alone Application:
-------------------------------------------
-> You just run a main() method like any regular Java program.
-> It includes an embedded server (e.g., Tomcat) in the application itself.
-> No need to install or configure any external server separately.

Note: Since Spring boot is stand alone application, we package the application as jar file


Spring Boot Workflow (Short Answer)
#####################################
You run the Spring Boot app ‚Üí main() executes
SpringApplication.run() starts the Spring context
Auto-configuration configures beans
Embedded Tomcat starts
Your controllers/services/repos load
Application becomes ready to receive requests



 What is @Service?
######################

####################

c. It makes your class eligible for dependency injection.

What is Status Code
###################
Status codes in Spring Boot are HTTP response codes (like 200, 400, 404, 500) sent by the server to indicate success, client error, or server error in a request.

Status Code: 201 It  indicates that a resource has been successfully created as a result of a POST request.

Status Code: 200 ‚Äì OK --> means The request was successful, and the server has returned the expected response.

Status Code: 404 ‚Äì Not Found --> means The resource the client is trying to access does not exist on the server.

Status Code : 500 ‚Äì Internal Server Error  means --> The server encountered an unexpected error while processing the request.
Causes:
NullPointerException
Database connection error

Status Code : 403 ‚Äì Forbidden  --> means The server understood the request, the resource exists, but the client is not allowed to access it. because User does NOT have required permissions 
This is an authorization error, not an authentication error.

What is the difference between PUT and PATCH
############################################
PUT and PATCH are HTTP methods used for updating resources in REST APIs, but they work differently.

PUT is used for a full update.
When we use PUT, we send the entire object, and the server replaces the existing data completely with the new one.
That means if any field is missing in the request body, it may be overwritten or set to null.

PATCH is used for a partial update.
With PATCH, we only send the fields we want to change, and the server updates only those specific values, leaving the remaining data unchanged.

//Executes for every request passing through the gateway
//Controls filter execution order
@Component
public class JwtAuthenticationFilter implements GlobalFilter, Ordered {
     //Used to verify JWT signature
    private static final String SECRET_KEY = "my-super-secret-key";

    // Public endpoints
    //These APIs do not require JWT
    //Login & registration must be public
    private static final List<String> openEndpoints = List.of(
            "/api/v1/auth/login",
            "/api/v1/auth/register-admin",
            "/api/v1/auth/register-user"
    );

    // Protected endpoint, only ROLE_ADMIN can access
    private static final Map<String, List<String>> protectedEndpoints =
            Map.of(
                    "/api/v1/data", List.of("ROLE_ADMIN") // Microservice1 protected endpoint
            );
     
    
    //This is the heart of the filter
    @Override
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
        
    	
    	//Used to check whether endpoint is:Public or Protected
        String path = exchange.getRequest().getURI().getPath();

        // 1Ô∏è‚É£ Allow public endpoints ‚úî
        //If request is for login or registration
        // Skip JWT validation
        // Forward request immediately
        if (openEndpoints.stream().anyMatch(path::startsWith)) {
            return chain.filter(exchange);
        }

        // 2Ô∏è‚É£ Token Required  // Check Authorization Header JWT must be sent in:authorization 
        if (!exchange.getRequest().getHeaders().containsKey(HttpHeaders.AUTHORIZATION)) {
            return onError(exchange, "Missing Authorization header", HttpStatus.UNAUTHORIZED);
        }

        String authHeader = exchange.getRequest().getHeaders().getFirst(HttpHeaders.AUTHORIZATION);
            
        //Validate Bearer Format ,Prevents invalid headers
        if (authHeader == null || !authHeader.startsWith("Bearer ")) {
            return onError(exchange, "Invalid Authorization format", HttpStatus.UNAUTHORIZED);
        }
         //Extract Token from header and removes Bearer prefix
        String token = authHeader.substring(7);

        // 3Ô∏è‚É£ Validate jwt token
        String role;
        try {
            role = JWT.require(Algorithm.HMAC256(SECRET_KEY))
                    .build()
                    .verify(token)
                    .getClaim("role")
                    .asString();

        } catch (Exception e) {
            return onError(exchange, "Invalid/Expired Token", HttpStatus.UNAUTHORIZED);
        }

        // 4Ô∏è‚É£ Role-based access check
        for (String endpoint : protectedEndpoints.keySet()) {
            if (path.startsWith(endpoint)) {

                List<String> allowedRoles = protectedEndpoints.get(endpoint);

                if (!allowedRoles.contains(role)) {
                    return onError(exchange, "Access Denied", HttpStatus.FORBIDDEN);
                }
            }
        }
        //Request forwarded to microservice
        return chain.filter(exchange);
    }
    //Error Handling Method ,401 Unauthorized,403 Forbidden
    private Mono<Void> onError(ServerWebExchange exchange, String err, HttpStatus status) {
        exchange.getResponse().setStatusCode(status);
        return exchange.getResponse().setComplete();
    }
     
    //Runs before most other filters
    @Override
    public int getOrder() {
        return -1;
    }
}
